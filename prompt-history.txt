สรุป Logic ที่ออกแบบ
=====================================================

Single Simulation Logic:
  1. รับ input: rolls (N), price, pity on/off, free roll on/off
  2. คำนวณ free rolls = floor(N / every) × grant
  3. total rolls = N + free rolls
  4. loop total rolls ครั้ง:
     a. ตรวจ pity: ถ้า pityCnt >= threshold-1 → force SSR, reset pityCnt
     b. ถ้าไม่ → rand = Math.random() × 100
        สะสม cumulative: SSR → SR → R → N จนถึง rand < cumulative
     c. เลือก item จาก pool ของ rarity นั้น (Math.random)
  5. รวม counts, itemCounts, rollLog

Free Roll Rule:
  free = floor(paidRolls / every) × grant
  → คำนวณจาก paid rolls เท่านั้น
  → free rolls ไม่สร้าง free rolls (ไม่ recursive)

Pity System:
  pityCnt เพิ่มทุก roll ที่ไม่ได้ SSR
  pityCnt reset เป็น 0 ทุกครั้งที่ได้ SSR
  ถ้า pityCnt >= threshold-1 → roll ถัดไปการันตี SSR

Monte Carlo (Player POV):
  loop simN ครั้ง:
    ใช้ paidRolls + freeRolls เป็น total rolls ต่อรอบ
    นับ SSR count ต่อรอบ
  คำนวณ P(SSR≥1), P(SSR=0), average, best, worst

CSV Export:
  Pull Result: Roll No., Rarity, Item Name, Cost Type, Cost
  POV Summary: Budget, Price, Paid, Free, Total, SimN, P1, P0, Avg, Best, Worst, Rates
  ทั้งคู่ใช้ BOM (﻿) + UTF-8 เพื่อรองรับภาษาไทยใน Excel
